## 二、关键发现

### 1. KlineRow 类型分裂
- `storage::KlineRow`（double 浮点价，time_series_cache.h:42）vs `event::KlineRow`（int32_t 定点价，kline_event.h:13）两套不兼容类型
- 设计文档明确"C++ 内部使用整数价格（1/10000 元）"
- WriteBuffer 用 event::KlineRow 写 StorageEngine 但引擎接受 storage::KlineRow — 类型不匹配

### 2. 存储链断裂
- `TimeSeriesStore::store_kline()` 只写缓存不写磁盘 — 崩溃丢数据
- `StorageEngine::query_kline()` 只查缓存 — cache miss 返回空
- 缓存无淘汰逻辑无预算控制 — 长时间运行 OOM
- DataIngestor 绕过 WriteBuffer 直接写 ColumnBlock — 无 WAL 保护

### 3. 策略链断裂
- EtcdClient + StrategyWatcher 代码已实现但 service_main 未集成
- StrategyWatcher 用 ID 查 name 注册表 — 查找失败
- 回测前缀未 watch

### 4. 阻塞原语遍地
- EventBus/ConfigManager/FactorComputer/CronScheduler/WsSession/WebSocketServer/Logger 等 13 个组件仍用 std::mutex/std::shared_mutex/std::condition_variable

### 5. 测试大量被删除
- storage_test.cc 从 1224 行删减到 236 行（87% 删除）
- ColumnBlock/DiskPersistence/WAL/WriteBuffer 测试全部丢失
